## Fase 1 — Infraestrutura e Módulos Fracos (Abril de 2026)

**Marco-alvo: M1**

### Pilar 4: Integração de Novatos

- [ ] **T-001** — Escrever `scripts/quickstart.sh`
  - Detecta o diretório `~/.claude/` (cria com confirmação do usuário se ausente)
  - Copia o primeiro slash command + CLAUDE.md + skill para a configuração do usuário
  - Adiciona linha de descoberta de agentes ao CLAUDE.md vinculando ao AGENT-INDEX.md
  - Idempotente: pula arquivos existentes com um aviso, nunca sobrescreve
  - Exibe os próximos passos ao concluir

- [ ] **T-002** — Criar `QUICKSTART.md` (guia visual "Primeiros 15 Minutos")
  - Passos anotados no terminal como blocos de código (evitar capturas de tela PNG onde possível; preferir ASCII ou Mermaid)
  - Documentar quais comandos produzem cada visual para que possam ser recapturados

- [ ] **T-003** — Adicionar badges de dificuldade a todos os 10 READMEs dos módulos
  - Formato: `> 🟢 **Iniciante** | ⏱ 30 minutos` no topo de cada README do módulo
  - Adicionar prévia de bullet "O que você vai construir" abaixo do badge

### Pilar 7: Currículo Vivo

- [ ] **T-004** — Criar `WHATS-NEW.md`
  - Formato: `## CC vX.X — DATA` com itens de bullet por mudança de recurso
  - Adicionar pelo menos uma entrada para a versão atual do CC

- [ ] **T-005** — Adicionar badges de versão a todos os 10 READMEs dos módulos
  - Formato: `> ✅ Verificado contra Claude Code **vX.X.X** · Última verificação: AAAA-MM-DD`
  - Adicionar `cc_version_verified` ao frontmatter de cada módulo

- [ ] **T-006** — Criar `.github/workflows/staleness-check.yml`
  - Agendamento: semanalmente às segundas-feiras às 09:00 UTC
  - Lê a data `last_verified` de cada módulo a partir do frontmatter (use `yq` ou script Python de 10 linhas)
  - Cria uma Issue no GitHub para qualquer módulo não verificado em 30+ dias
  - Pula se já existir uma issue aberta com o mesmo título

### Pilar 6: Qualidade de Conteúdo (CLAUDE.md)

- [ ] **T-007** — Criar `CLAUDE.md` raiz para o projeto
  - Demonstra boas práticas como a própria configuração do projeto
  - Documenta convenções de contribuição, estrutura de módulos e workflow de manutenção

### Pilar 2: Índice para Agentes de IA (Gerador)

- [ ] **T-008** — Escrever `scripts/build-agent-index.py`
  - Lê todos os 10 arquivos README dos módulos
  - Extrai: cabeçalhos (→ tópicos), blocos de código (→ exemplos com tags de linguagem), tabelas (→ dados de referência)
  - Gera `agent-manifest.json` com schema definido no plano CEO
  - Gera `AGENT-INDEX.md` (resumo legível por humanos)
  - Valida que as referências apontam para arquivos reais

### Módulo 08-checkpoints: Passagem Completa Aprofundada (311 → 800+ linhas)

- [ ] **T-009** — Reescrever a introdução do módulo com um cenário do mundo real
  - Exemplo: "Sua refatoração acabou de quebrar 3 coisas. Veja como os checkpoints te salvam..."

- [ ] **T-010** — Adicionar árvore de decisão de estratégia de checkpoint (fluxograma Mermaid)
  - "Quero fazer X" → seguir setas → chegar ao padrão de checkpoint

- [ ] **T-011** — Adicionar 3 templates de workflow
  - Workflow de refatoração segura
  - Testes A/B com checkpoints
  - Workflow de recuperação de desastres

- [ ] **T-012** — Adicionar seção de integração com o modo de planejamento

- [ ] **T-013** — Adicionar diagrama Mermaid para o ciclo de vida do checkpoint

- [ ] **T-014** — Adicionar 2+ blocos "Tente Agora" com comandos verificáveis
  - Cada um: frase de contexto, comando para executar, descrição da saída esperada

- [ ] **T-015** — Adicionar seção "Padrões e Receitas" com 2+ padrões nomeados
  - Cada padrão: problema, código de solução, quando usar, quando NÃO usar

### Módulo 06-hooks: Passagem Completa Aprofundada

- [ ] **T-016** — Reescrever a introdução do módulo com um cenário do mundo real
  - Exemplo: "Seu CI detectou um erro de lint após 20 minutos. Hooks detectam em 2 segundos..."

- [ ] **T-017** — Adicionar árvore de decisão de hooks (fluxograma Mermaid)
  - "Quero acionar X quando Y acontecer" → seguir setas → chegar ao tipo de hook

- [ ] **T-018** — Adicionar 5 exemplos completos de hooks com scripts completos (~80 linhas cada)
  1. Auto-formatação ao escrever (prettier/black)
  2. Varredura de segurança no commit (trufflehog/bandit)
  3. Executor de testes ao alterar arquivo
  4. Notificação ao encerrar sessão
  5. Validação de prompt (bloquear padrões perigosos)

- [ ] **T-019** — Adicionar 2+ blocos "Tente Agora" com comandos verificáveis

- [ ] **T-020** — Adicionar seção "Padrões e Receitas" com 2+ padrões nomeados

**Checkpoint M1: infraestrutura online, 2 módulos mais fracos totalmente atualizados**
