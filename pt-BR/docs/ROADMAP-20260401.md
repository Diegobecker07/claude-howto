# Roadmap: claude-howto 2026–2027

> Abril de 2026 – Março de 2027 · Base de Conhecimento de Duas Camadas | Plano completo: `TASKS-20260401.md`

---

## Visão

Transformar o claude-howto de um repositório de tutoriais estático em um **sistema de conhecimento vivo para dois públicos**:

- **Para humanos** — aprendizado interativo e baseado em cenários com dificuldade progressiva, árvores de decisão e padrões nomeados que especialistas favoritam
- **Para agentes de IA** — índice de metadados estruturado que agentes consultam antes de realizar tarefas no Claude Code, tornando este repositório infraestrutura, não apenas conteúdo

Nenhum concorrente tem agentes de IA como público primário. Esta é a vantagem competitiva.

---

## Os 7 Pilares

| # | Pilar | O que entrega |
|---|-------|---------------|
| P1 | Camada Divertida | Introduções de cenários + blocos "Tente Agora" em cada módulo |
| P2 | Índice para Agentes de IA | `agent-manifest.json` gerado + `AGENT-INDEX.md` + skill de busca |
| P3a | Referência Especializada (no módulo) | Árvores de decisão + padrões nomeados por módulo |
| P3b | Referência Especializada (entre módulos) | `RECIPES.md` — workflows compostos com múltiplos recursos |
| P4 | Integração de Novatos | `quickstart.sh` + `QUICKSTART.md` + badges de dificuldade |
| P5 | Vitrine Comunitária | `COMMUNITY-PROJECTS.md` — projetos de usuários curados |
| P6 | Qualidade de Conteúdo | Expandir módulos mais fracos; `CLAUDE.md` do projeto |
| P7 | Currículo Vivo | `WHATS-NEW.md` + badges de versão + ação CI semanal de desatualização |

## Marcos

### M1 — Infraestrutura Online · Fim de Abril de 2026

**O que é entregue:**
- `scripts/quickstart.sh` — configuração com um comando para novos usuários (idempotente)
- `QUICKSTART.md` — guia visual "Primeiros 15 Minutos"
- Badges de dificuldade + prévias "O que você vai construir" em todos os 10 módulos
- `WHATS-NEW.md` + badges de versão em todos os módulos
- `.github/workflows/staleness-check.yml` — issue semanal às segundas se módulo não verificado em 30+ dias
- `CLAUDE.md` raiz — configuração própria do projeto como exemplo de boas práticas
- `scripts/build-agent-index.py` — gerador que lê todos os 10 módulos → gera `agent-manifest.json` + `AGENT-INDEX.md`
- **06-hooks** — passagem completa aprofundada: 5 scripts de hook completos, árvore de decisão, Tente Agora, padrões
- **08-checkpoints** — passagem completa aprofundada: 311 → 800+ linhas, 3 templates de workflow, árvore de decisão, padrões

**Por que começar aqui:** A infraestrutura beneficia todas as fases futuras. Hooks e checkpoints são os módulos mais fracos e com maior chance de perder novos visitantes.

### M4 — Camada de Agentes Online · Fim de Setembro de 2026

**O que é entregue:**
- `agent-manifest.json` final cobrindo 100% dos módulos (gerado a partir do conteúdo completo)
- `AGENT-INDEX.md` vinculado a partir do `README.md`
- `skills/claude-howto-lookup/SKILL.md` — skill leve de agente que consulta o manifest
- `RECIPES.md` — 5+ workflows compostos (schema: nome, módulos-usados, problema, solução, resultado esperado)
- `COMMUNITY-PROJECTS.md` — lista curada estática com formato de envio via PR

**Por que setembro:** O índice de agentes só é significativo quando todos os 10 módulos estiverem com conteúdo completo.

---

### M5 — Auditoria de Versão Completa · Fim de Novembro de 2026

**O que é entregue:**
- Auditoria completa de versão: todos os 10 módulos verificados contra a versão atual do CC
- Frontmatter `cc_version_verified` atualizado + badges de versão em todos os lugares
- `RECIPES.md` expandido para 8+ receitas com base nos padrões observados pela comunidade
- Discussão do GitHub fixada: "Compartilhe seus workflows com o Claude Code" — coleta de sinais de uso por agentes

---

### M6 — Sistema Autossustentável · Fim de Março de 2027

**O que é entregue / executado continuamente:**
- Skill `/docs-sync-claude-code` executada após cada lançamento do CC → `WHATS-NEW.md` atualizado
- Regressão CI do manifest de agentes: cobertura de 100% dos módulos imposta
- `RECIPES.md` com 10+ receitas
- `COMMUNITY-PROJECTS.md` crescendo organicamente
- Sinal de uso por agentes avaliado → se validado, promover a skill de busca (marketing, registro asm)
