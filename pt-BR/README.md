## O Problema

Você instalou o Claude Code. Rodou alguns prompts. E agora?

- **A documentação oficial descreve recursos — mas não mostra como combiná-los.** Você sabe que slash commands existem, mas não como encadeá-los com hooks, memória e subagentes em um workflow que realmente economiza horas.
- **Não há um percurso de aprendizado claro.** Você deve aprender MCP antes de hooks? Skills antes de subagentes? Você acaba lendo tudo superficialmente e não dominando nada.
- **Os exemplos são básicos demais.** Um slash command "hello world" não te ajuda a construir um pipeline de revisão de código em produção que usa memória, delega a agentes especializados e executa varreduras de segurança automaticamente.

Você está deixando 90% do poder do Claude Code na mesa — e não sabe o que não sabe.

## FAQ

**É gratuito?**
Sim. Licenciado MIT, gratuito para sempre. Use em projetos pessoais, no trabalho, na sua equipe — sem restrições além de incluir o aviso de licença.

**É mantido ativamente?**
Sim. O guia é sincronizado com cada lançamento do Claude Code. Versão atual: v2.3.0 (abril de 2026), compatível com Claude Code 2.1+.

**Como isso é diferente da documentação oficial?**
A documentação oficial é uma referência de recursos. Este guia é um tutorial com diagramas, templates prontos para produção e um percurso de aprendizado progressivo. Eles se complementam — comece aqui para aprender, consulte a documentação quando precisar de detalhes específicos.

**Quanto tempo leva para percorrer tudo?**
11-13 horas para o percurso completo. Mas você terá valor imediato em 15 minutos — basta copiar um template de slash command e experimentar.

**Posso usar com Claude Sonnet / Haiku / Opus?**
Sim. Todos os templates funcionam com Claude Sonnet 4.6, Claude Opus 4.6 e Claude Haiku 4.5.

**Posso contribuir?**
Com certeza. Veja [CONTRIBUTING.md](../CONTRIBUTING.md) para as diretrizes. Damos boas-vindas a novos exemplos, correções de bugs, melhorias na documentação e templates da comunidade.

**Posso ler isso offline?**
Sim. Execute `uv run scripts/build_epub.py` para gerar um e-book EPUB com todo o conteúdo e diagramas renderizados.

<summary>06. Hooks</summary>

**Localização**: [06-hooks/](../06-hooks/)

**O quê**: Comandos shell orientados a eventos que executam automaticamente em resposta a eventos do Claude Code

**Exemplos**:
- `format-code.sh` - Auto-formatar código antes de escrever
- `pre-commit.sh` - Executar testes antes de commits
- `security-scan.sh` - Varrer por problemas de segurança
- `log-bash.sh` - Registrar todos os comandos bash
- `validate-prompt.sh` - Validar prompts do usuário
- `notify-team.sh` - Enviar notificações em eventos

**Instalação**:
```bash
mkdir -p ~/.claude/hooks
cp 06-hooks/*.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh
```

Configure hooks em `~/.claude/settings.json`:
```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Write",
      "hooks": ["~/.claude/hooks/format-code.sh"]
    }],
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": ["~/.claude/hooks/security-scan.sh"]
    }]
  }
}
```

**Uso**: Hooks executam automaticamente em eventos

**Tipos de Hook** (4 tipos, 25 eventos):
- **Tool Hooks**: `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `PermissionRequest`
- **Session Hooks**: `SessionStart`, `SessionEnd`, `Stop`, `StopFailure`, `SubagentStart`, `SubagentStop`
- **Task Hooks**: `UserPromptSubmit`, `TaskCompleted`, `TaskCreated`, `TeammateIdle`
- **Lifecycle Hooks**: `ConfigChange`, `CwdChanged`, `FileChanged`, `PreCompact`, `PostCompact`, `WorktreeCreate`, `WorktreeRemove`, `Notification`, `InstructionsLoaded`, `Elicitation`, `ElicitationResult`

<summary>08. Checkpoints e Retrocesso</summary>

**Localização**: [08-checkpoints/](../08-checkpoints/)

**O quê**: Salvar o estado da conversa e retroceder a pontos anteriores para explorar abordagens diferentes

**Conceitos-Chave**:
- **Checkpoint**: Snapshot do estado da conversa
- **Rewind**: Retornar ao checkpoint anterior
- **Branch Point**: Explorar múltiplas abordagens a partir do mesmo checkpoint

**Uso**:
```
# Checkpoints são criados automaticamente a cada prompt do usuário
# Para retroceder, pressione Esc duas vezes ou use:
/rewind

# Em seguida, escolha entre cinco opções:
# 1. Restaurar código e conversa
# 2. Restaurar conversa
# 3. Restaurar código
# 4. Resumir daqui
# 5. Deixa pra lá
```

**Casos de Uso**:
- Experimentar diferentes abordagens de implementação
- Recuperar-se de erros
- Experimentação segura
- Comparar soluções alternativas
- Testes A/B de designs diferentes

```markdown
# Usa: Skills + Subagents + Memory

Usuário: "Gere documentação de API para o módulo de autenticação"

Claude:
1. Carrega memória do projeto (padrões de documentação)
2. Detecta requisição de geração de documentação
3. Invoca automaticamente a skill doc-generator
4. Delega ao subagente api-documenter
5. Cria documentação abrangente com exemplos
```

### Implantação DevOps

```markdown
# Usa: Plugins + MCP + Hooks

Usuário: /deploy production

Claude:
1. Executa hook pré-deploy (valida ambiente)
2. Delega ao subagente deployment-specialist
3. Executa implantação via Kubernetes MCP
4. Monitora o progresso
5. Executa hook pós-deploy (verificações de saúde)
6. Reporta o status
```

<summary>Boas Práticas</summary>

### Faça
- Comece simples com slash commands
- Adicione recursos incrementalmente
- Use memória para padrões de equipe
- Teste configurações localmente primeiro
- Documente implementações personalizadas
- Versione configurações do projeto
- Compartilhe plugins com a equipe

### Não Faça
- Não crie recursos redundantes
- Não codifique credenciais diretamente
- Não pule a documentação
- Não complique tarefas simples demais
- Não ignore boas práticas de segurança
- Não commite dados sensíveis

</details>

<details>
<summary>Solução de Problemas</summary>

### Recurso Não Está Carregando
1. Verifique a localização e o nome do arquivo
2. Verifique a sintaxe do frontmatter YAML
3. Verifique as permissões do arquivo
4. Revise a compatibilidade de versão do Claude Code

### Falha na Conexão MCP
1. Verifique as variáveis de ambiente
2. Verifique a instalação do servidor MCP
3. Teste as credenciais
4. Revise a conectividade de rede

### Subagente Não Está Delegando
1. Verifique as permissões de ferramentas
2. Verifique a clareza da descrição do agente
3. Revise a complexidade da tarefa
4. Teste o agente de forma independente

</details>

<details>
<summary>Testes</summary>

Este projeto inclui testes automatizados abrangentes:

- **Testes Unitários**: Testes Python usando pytest (Python 3.10, 3.11, 3.12)
- **Qualidade de Código**: Lint e formatação com Ruff
- **Segurança**: Varredura de vulnerabilidades com Bandit
- **Verificação de Tipos**: Análise de tipos estática com mypy
- **Verificação de Build**: Teste de geração de EPUB
- **Rastreamento de Cobertura**: Integração com Codecov

```bash
# Instalar dependências de desenvolvimento
uv pip install -r requirements-dev.txt

# Executar todos os testes unitários
pytest scripts/tests/ -v

# Executar testes com relatório de cobertura
pytest scripts/tests/ -v --cov=scripts --cov-report=html

# Executar verificações de qualidade de código
ruff check scripts/
ruff format --check scripts/

# Executar varredura de segurança
bandit -c pyproject.toml -r scripts/ --exclude scripts/tests/

# Executar verificação de tipos
mypy scripts/ --ignore-missing-imports
```

Testes executam automaticamente em cada push para `main`/`develop` e em cada PR para `main`. Veja [TESTING.md](../.github/TESTING.md) para informações detalhadas.

</details>
